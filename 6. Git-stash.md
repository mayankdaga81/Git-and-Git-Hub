# 📦 Git Stash – Quick Revision Notes

---

## 🔹 Why Do We Need `git stash`?

If you have uncommitted changes in your current branch:

❌ You cannot switch branches safely.  
Git will prevent checkout if changes may conflict.

👉 Solution: Use **stash** to temporarily store changes.

---

# 🧠 What is `git stash`?

- Acts like a **temporary shelf**.
- Stores uncommitted changes.
- Clears your working directory.
- Allows you to switch branches safely.
- Changes are NOT committed.

---

# 1️⃣ Stash Your Changes

```bash
git stash
```

- Saves:
  - Modified tracked files
- Reverts working directory to last commit state.

Now you can switch branches:

```bash
git switch master
```

---

# 2️⃣ Bring Back Stashed Changes

### 🔹 `git stash pop`

```bash
git stash pop
```

- Applies the latest stash.
- Removes it from stash list.
- Restores changes to current branch.

---

# 3️⃣ View Stash List

```bash
git stash list
```

Example output:

```
stash@{0}: WIP on footer
stash@{1}: WIP on nav-bar
```

- Stores multiple stashes.
- Each stash has an index.

---

# 4️⃣ Apply Specific Stash

```bash
git stash apply stash@{0}
```

- Applies selected stash.
- Does NOT remove it from stash list.

---

# 🔥 Important Difference

- `git stash pop` → Apply + Remove
- `git stash apply` → Apply only (keeps in list)

---

# ⚠️ Important Notes (Interview Focus)

- Stash is NOT branch-specific.
  - You can stash in `footer` branch.
  - Pop it in `master`.
- Can cause conflicts when applying.
- Only tracked files are stashed by default.
- Stash does NOT create a commit.
- Use carefully — can get confusing with multiple stashes.

---

# 🧩 Useful Extra Commands

### 🔹 Stash including untracked files

```bash
git stash -u
```

### 🔹 Delete a stash

```bash
git stash drop stash@{0}
```

### 🔹 Clear all stashes

```bash
git stash clear
```

---

# 🧠 Simple Flow

```
Make changes → git stash → switch branch → git stash pop
```

---

💡 `git stash` = Temporary storage for uncommitted work.
