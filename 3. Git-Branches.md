# 🌿 Git Branches – Quick Revision Notes

---

## 1️⃣ Check Current Branch

### 🔹 `git branch`

- Shows all branches.
- The current branch is marked with `*`.

---

## 2️⃣ Create a New Branch

### 🔹 `git branch branch_name`

- Creates a new branch named `branch_name`.
- Does NOT switch to it automatically.

---

## 3️⃣ Switch Branch

### 🔹 `git checkout branch_name`

- Switches to the specified branch.

### 🔹 `git switch branch_name`

- Also switches to the specified branch.
- Cleaner and more modern alternative to `checkout`.

---

## 4️⃣ Create + Switch in One Command

### 🔹 `git checkout -b branch_name`

- Creates a new branch.
- Immediately switches to it.

> If you use `git checkout branch_name` without `-b`, it will only switch branches (it will not create a new one).

---

## 5️⃣ Delete a Branch

### 🔹 `git branch -d branch_name`

- Deletes the specified branch.
- Safe delete (won’t delete if not merged).

---

# 🌿 Branch Workflow Example

```bash
git branch feature-login
git switch feature-login
# OR
git checkout -b feature-login
```

---

💡 Branches are used to work on features, bug fixes, or experiments without affecting the main branch.
