# 🔎 Git Diff – Quick Revision Notes

---

## 🔹 What is `git diff`?

- Shows the difference between changes in files.
- Compares:
  - Working Directory ↔ Staging Area
  - Staging Area ↔ Last Commit
  - Commit ↔ Commit

---

# 🧠 Important Clarification

❌ Default `git diff` does NOT compare two branches directly.  
✅ It compares file differences between states (working, staging, commits).

(You *can* compare branches explicitly, but default `git diff` does not do that automatically.)

---

# 1️⃣ Basic Usage

### 🔹 Check unstaged changes

```bash
git diff
```

- Shows difference between:
  - Working Directory
  - Staging Area

---

### 🔹 Check staged changes

```bash
git diff --staged
# or
git diff --cached
```

- Shows difference between:
  - Staging Area
  - Last Commit

---

# 🧩 Symbols in `git diff`

When you see output:

```
--- a/index.html
+++ b/index.html
```

- `a` → Old version (Version X)
- `b` → New version (Version Y)

⚠️ Important:
- `---` does NOT mean line is deleted.
- `+++` does NOT mean line is added.
- They just represent file versions.

---

Inside the diff:

- Lines starting with `-` → Removed from old version
- Lines starting with `+` → Added in new version
- No symbol → Unchanged context

---

# 🧪 Example Scenario

1. Modify `index.html`
2. Stage it:

```bash
git add index.html
```

3. Check staged difference:

```bash
git diff --staged
```

It will show differences between:
- Last committed version
- Current staged version

---

# 2️⃣ Compare Two Commits

### Step 1 – Get commit IDs

```bash
git log --oneline
```

Example output:

```
bae3424 Updated navbar
2df06d7 Added footer
```

### Step 2 – Compare commits

```bash
git diff bae3424 2df06d7
```

- Shows difference between those two commits.

---

# 🔥 Additional Important Commands

### 🔹 Compare specific file between commits

```bash
git diff bae3424 2df06d7 index.html
```

---

### 🔹 Compare two branches

```bash
git diff master nav-bar
```

- Shows difference between branches.

---

# ⚠️ Important Notes (Interview Focus)

- `git diff` does NOT change anything — it only shows differences.
- Useful before committing to review changes.
- Helps in debugging.
- Helps understand what exactly changed.
- Always check diff before pushing to production.
